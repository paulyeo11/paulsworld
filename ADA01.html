Path A — Step by Step (PC + IB Gateway)路径A — 逐步操作（电脑 ＋ IB Gateway）

1
Double-click refresh-ibkr.bat双击 refresh-ibkr.bat
Sits in the project folder. It first pulls the latest scripts from GitHub, then checks whether IB Gateway is already connected on port 4001.该文件位于项目文件夹中。运行时会先从GitHub拉取最新脚本，再检查IB Gateway是否已在4001端口连接。
2
Log in if IB Gateway isn't running若IB Gateway尚未运行，请登入
The script auto-launches the IB Gateway login window if needed. Enter your ID + password and approve 2FA on your phone — the script waits up to 3 minutes and continues automatically once connected.如有需要，脚本会自动打开IB Gateway登入窗口。输入账号密码并在手机上批准双重验证——脚本最多等待3分钟，连接成功后会自动继续。
3
Script fetches positions脚本拉取持仓数据
ibkr_positions.py pulls live positions, NAV and P&L straight from IB Gateway, then writes and pushes the updated data to GitHub — no copy/paste, no manual upload.ibkr_positions.py 直接从IB Gateway拉取实时持仓、资产净值与损益，然后将更新后的数据写入并推送到GitHub——无需复制粘贴，无需手动上传。
4
T29 updates liveT29自动更新
The GitHub push triggers Vercel/Netlify's auto-deploy — Option IC Deskboard shows the fresh numbers within a couple of minutes. No further action needed.推送到GitHub会触发Vercel/Netlify的自动部署——期权铁鹰仪表板会在几分钟内显示最新数据，无需其他操作。

Path B — Step by Step (Phone Only, via Google Drive)路径B — 逐步操作（仅用手机，透过Google Drive）

1
Statement lands in the "IBKR" Drive folder对账单存入「IBKR」Drive文件夹
Usually happens automatically — a Google Apps Script runs daily (~7–8pm SGT), finds IBKR emails with attachments in Gmail, and auto-saves new ones into Paul's Drive. Paul can also save/overwrite the export there himself any time — no link or attachment needs to be pasted into chat.通常会自动完成——一个Google Apps Script每天（约新加坡时间晚上7–8点）运行，在Gmail中查找带附件的IBKR邮件，并自动将新附件存入Paul的Drive。Paul也可以随时自行在该处保存／覆盖导出文件——无需在聊天中粘贴链接或附件。
2
Say the trigger phrase说出触发指令
Tell David/Claude: "update IBKR data" (or "update my IBKR raw data" / "check the IBKR folder in Drive") — any of these are treated the same way.告诉David/Claude：「更新IBKR数据」（或「更新我的IBKR原始数据」／「查看Drive里的IBKR文件夹」）——这些说法都会以相同方式处理。
3
Claude reads the raw file & cross-checks itClaude读取原始文件并交叉核对
The Google Drive tools open the latest file, and every figure is checked against at least one other section that should reconcile (e.g. Account Overview NAV vs. Allocation totals vs. the monthly NAV chart) before it's written anywhere. If a number can't be confidently cross-checked, it's left unchanged rather than guessed.Google Drive工具会打开最新文件，每个数字在被写入之前，都会与至少一个理应对得上的其他部分交叉核对（例如账户概览的资产净值 vs. 资产配置总额 vs. 月度净值图表）。若某个数字无法确认，则维持不变，绝不猜测。
4
T29 gets updated & pushedT29获得更新并推送
options-breadboard.html (T29) is edited with the verified figures and pushed straight to GitHub — live on both Vercel and Netlify within minutes.options-breadboard.html（T29）会以核实后的数字更新并直接推送到GitHub——几分钟内即在Vercel与Netlify同步上线。
5
A dated recap article is written生成一篇带日期的更新摘要文章
A new IB##.html entry is added summarizing what changed — Ending NAV vs. last update, unrealized P&L moves, deposits and any position changes — in plain bilingual language, and linked into the IBKR Update Log section on the homepage.系统会新增一篇 IB##.html 文章，以简明双语总结变化——期末资产净值对比上次更新、未实现盈亏变动、存款与持仓变化——并链接到首页的IBKR更新日志区块。
