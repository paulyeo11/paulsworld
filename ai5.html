🔑 The Secret — A GitHub Personal Access Token

For Claude to access your GitHub, you need to give it a special password called a Personal Access Token (PAT). This is a long code that gives Claude permission to read and write your files. You create it once on GitHub and share it with Claude — that's all.

💡 Think of the token like a spare key to your house. You give it to a trusted helper (Claude) so they can come in and do the work — without you having to be there to open the door every time.

📋 How to Create Your GitHub Token — Step by Step

Step 1
Go to github.com and sign in to your account.
Step 2
Click your profile photo (top-right corner) → then click Settings.
Step 3
Scroll all the way down the left sidebar and click Developer settings.
Step 4
Click Personal access tokens → then Tokens (classic) → then Generate new token (classic).
Step 5
Give it a name like claude-website-access. Under Expiration, choose No expiration so it never expires.
Step 6
Under Scopes, tick the repo checkbox. This gives Claude permission to read and write your files. No other boxes needed.
Step 7
Click Generate token. GitHub shows you the token — it starts with ghp_. Copy it immediately — GitHub will never show it again.
Step 8
Save the token somewhere safe — in a notes app or password manager. Then paste it into Claude's user preferences so Claude always has access.
⚠️ Important — Keep Your Token Private

Your GitHub token is like your password. Never share it publicly, never post it in a public chat or forum. Only share it with Claude in your private conversation. If you ever think it has been exposed, go back to GitHub Settings and delete it, then create a new one.

🔄 How the Auto-Push Works Behind the Scenes

You don't need to understand the technical details to use this — but for those who are curious, here is what happens every time Claude pushes a file to your GitHub:

1
Claude writes the HTML file The complete file is created with all your standard rules — Home button, view counter, correct colours and fonts.
2
Claude calls the GitHub API Using your token, Claude sends the file directly to GitHub's servers using a secure API call — like a digital courier delivering a package.
3
GitHub saves the file with a commit GitHub creates a permanent record of the change — called a commit — with a unique SHA code like 532cf8a3. This is proof the file was saved.
4
Vercel detects the change and goes live Your website hosting (Vercel) automatically detects the new file on GitHub and publishes it to your live website within about 30 seconds.

📌 Key Things to Remember

After months of using Claude to manage my website, here are the most important lessons I have learned:

✅
Always confirm the commit SHA After every push, Claude gives you a SHA code. This confirms the file was saved successfully. If you don't see it, ask Claude to confirm.
✅
Visit the live page after every push Go to paulsworld.vercel.app and check the new or updated page looks correct. Take 30 seconds to verify — it saves hours of fixing later.
✅
Keep your image folders backed up locally Claude can push HTML files but cannot push image folders automatically. Always keep your photos backed up on your computer and Google Drive.
✅
Tell Claude your token once — save it in preferences Put your GitHub token in Claude's User Preferences so every conversation has access without you having to repeat it.
