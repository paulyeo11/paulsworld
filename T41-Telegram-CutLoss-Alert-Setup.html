Why This Exists为什么要做这个

Table 29's Cut-Loss tab already shows a 🟢/🟡/🔴 gap to each stock's low/high band — see T28: Cut-Loss Alerts. The gap is only useful the moment I open the page. A single unusually volatile day could push a price through a band while I'm asleep, traveling, or just not looking — so I built a background check that watches the same bands and messages me directly, without needing the dashboard open.Table 29的「止损」页签已能显示每支股票距离上下限止损区间的🟢/🟡/🔴差距——参见 T28：止损提醒。但这个差距只有在我打开页面那一刻才有用。若在我睡觉、出门或没留意时刚好出现异常剧烈波动的一天，价格就可能悄悄跌破或突破区间——因此我做了一个背景检查，能自动盯着同样的区间，并直接发消息通知我，不需要打开面板。

Setup Steps (One-Time)设置步骤（一次性）

1
Create a Telegram bot创建Telegram机器人
Message @BotFather on Telegram → /newbot → choose a name and a username ending in "bot". BotFather replies with a bot token (a long string) — this is the bot's password, keep it private.在Telegram上给@BotFather发消息 → /newbot → 设定一个名称与以「bot」结尾的用户名。BotFather会回复一个机器人令牌（一长串字符）——这相当于机器人的密码，务必保密。
2
Find my chat ID找出我的聊天ID
Rather than fiddling with API calls, the easy way: message the public bot @userinfobot — it instantly replies with my numeric Telegram ID. That number is the "chat ID" the alert needs to know where to send messages.与其手动调用API，更简单的方法是：给公开机器人@userinfobot发消息——它会立刻回复我的Telegram数字ID。这个数字就是提醒系统用来知道该往哪里发消息的「聊天ID」。
3
Add both as GitHub repo secrets将两者添加为GitHub仓库密钥
Repo → Settings → Secrets and variables → Actions → New repository secret. Add TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID. GitHub encrypts these — nobody, including me looking at the repo later, can read the value back out.仓库 → Settings → Secrets and variables → Actions → New repository secret。添加 TELEGRAM_BOT_TOKEN 与 TELEGRAM_CHAT_ID 两个密钥。GitHub会将其加密——包括我自己日后查看仓库时，也无法再读出原始值。
4
Write the check + workflow编写检查脚本与工作流
A small script fetches live prices, compares them to the bands, and calls the Telegram sendMessage API when a new breach appears. A GitHub Actions workflow file runs that script on a schedule.一段小脚本负责抓取实时价格、与区间比对，并在出现新的止损触发时调用Telegram的sendMessage接口。一个GitHub Actions工作流文件负责按排程运行该脚本。
5
Test with a manual run手动测试一次
The workflow has a manual "Run workflow" trigger with a checkbox to force a test message — regardless of whether anything has actually breached — so delivery could be confirmed before trusting the schedule.该工作流设有手动的「Run workflow」触发选项，附带一个勾选框可强制发送一则测试消息——无论当下是否真的有止损触发——藉此在信任排程之前先确认消息确实能送达。
✅ Confirmed working — test message received on Telegram已确认可正常运作——已在Telegram收到测试消息

What I Get Alerted On我会在什么情况下收到提醒

  • Runs automatically every hour, Monday–Friday — no need to open Table 29周一至周五每小时自动运行——无需打开Table 29
  • Alerts once per breach: if a symbol drops through its low band or rises through its high band, I get a message the first time — then it stays quiet while still breached同一次止损只提醒一次：股票若跌破下限或突破上限，第一次会立即收到消息——之后即使仍处于触发状态也不会重复提醒
  • Alerts again if the price clears the band and later breaches it a second time若价格回到区间内后再度触及止损，会再次收到提醒
⚠️ This is a coarse hourly check, not real-time — a fast intraday move could breach and recover between checks without ever triggering an alert.这是每小时一次的粗略检查，并非实时——若股价在两次检查之间快速触及又回弹，可能不会触发任何提醒。
