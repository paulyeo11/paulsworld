Why This Page Exists

Files were accidentally deleted from GitHub twice — including image folders that were hard to recover. These rules prevent it from happening again.

🚫

NEVER Delete Files on GitHub

The #1 danger

  • ❌Never delete any file or folder on GitHub — not even "old" ones.
  • ❌Deleting an image folder removes ALL photos inside it permanently.
  • ✅To retire a page, rename it with _old (e.g. s3_old.html) instead of deleting.
  • ✅Ask yourself: "Do I have a local backup of this?" — If no, do NOT delete.
🖼️

Always Back Up Image Folders

Images are the hardest to recover

  • 📁Every article has its own image folder (e.g. img_macritchie/, img_swiss/). Keep copies on your computer.
  • ⚠️Vercel backup does NOT restore image folders — only HTML files. You must restore images manually.
  • ☁️After uploading new photos to GitHub, immediately save a copy to Google Drive or USB.
🔍

Before Making Any Change

Simple 3-step habit

  • 1️⃣Save a copy of the current file to your computer before editing.
  • 2️⃣Do one change at a time — confirm it works before doing the next one.
  • 3️⃣After any AI push, visit the live page to confirm everything looks correct.
💾

Keep 3 Copies Always

GitHub + Local Computer + Google Drive

  • 🔵GitHub — your live website files (main copy).
  • 🟢Local Computer — folder on your desktop with ALL HTML files and image folders. Update weekly.
  • 🟡Google Drive / USB — upload everything once a month as a third backup.
🤖

Safe AI (Claude) Usage

Quick rules when using AI on your site

  • ✅Always ask AI to confirm the commit SHA after every push — this proves the file saved.
  • ✅Tell AI: "Never delete files — only create or update."
  • 🔒Your GitHub token (ghp_...) is private — never share it with anyone.

🆘 If Something Goes Wrong

1
STOP — don't make more changes. More edits make recovery harder.
2
Check GitHub commit history to find the last working version of the file.
3
Restore HTML from GitHub history or your local backup.
4
Restore images from LOCAL backup only — GitHub history does not save image folders. Re-upload them manually.
5
Test every page on the live site before considering the job done.
