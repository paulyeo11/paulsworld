TUT01: How to Set Up GitHub, Vercel, Notion & Gmail — The Foundation Behind This Whole System

Connect the four free tools that let Claude build, publish, remember, and act for you连接四个免费工具，让Claude为你构建、发布、记忆和行动

Paul Yeo  ·  June 2026  ·  Tutorials Series

Every guide in this Tutorials series — the travel agent, the investment agent, the travel book — assumes one thing is already in place: a simple foundation of free accounts that let Claude actually do things, not just chat. This is that foundation, explained one tool at a time.

None of this requires coding experience. I set all four of these up after I retired at 67, one account at a time, over a few weeks — not in one sitting. Take it slowly and get each one working before moving to the next.

· · ·

Step 1 — GitHub: Where Your Files Live

GitHub is a free place to store files with a complete history of every change. Think of it as a filing cabinet that never loses a version and that other tools (like Vercel) can read directly.

Getting Started with GitHub

1
Create a free account Go to github.com and sign up with your email. Choose a username you are comfortable sharing publicly if your site will be public.
2
Create a repository A repository ("repo") is one project's folder. Click "New repository," give it a name, and choose Public if you want a free website later.
3
Let Claude push for you If you use Claude Code, it can create files and push changes to your repo directly once it's authenticated. Every push is a saved, recoverable version of your work.
· · ·

Step 2 — Vercel: Auto-Publish Your Website

Vercel takes whatever is in your GitHub repository and turns it into a live website automatically — no separate "upload" step required.

  • Go to vercel.com and sign up using your GitHub account — this links the two automatically.
  • Click "Import Project" and select your GitHub repository.
  • Vercel deploys it instantly and gives you a live URL (mine is paulsworld.netlify.app).
  • From then on, every time a new file is pushed to your repo's main branch, Vercel republishes the site automatically — usually within a minute or two.

This is the exact pipeline behind this website: I (or Claude on my behalf) push a file to GitHub, and Vercel makes it live without me touching a server.

· · ·

Step 3 — Notion: Your Second Brain

Notion is where I keep everything Claude needs to remember about my life — travel plans, project notes, ideas — organised so Claude can read (and sometimes write) directly through its API.

Getting Started with Notion

1
Create a free account Sign up at notion.so. Start with a simple structure — even just a few pages for Projects, Areas, Resources, and Archive (the "PARA" method) is enough to begin.
2
Create an integration token In Notion's settings under "Connections," create an internal integration. This gives you a private API key that lets Claude read and write specific pages you choose to share with it.
3
Share pages with the integration Open each page you want Claude to access and use "Share" to connect your integration. Claude can only see pages you have explicitly shared — nothing else.
· · ·

Step 4 — Gmail & Calendar: Let Claude Check Your Inbox

Connecting Gmail and Google Calendar lets Claude search emails, draft replies, and check your schedule — without you ever typing your password into a chat.

⚠️ Never type your password into a chat

The right way to connect Gmail is through an official MCP connector or Claude's built-in integrations — never by pasting your email password or app password directly into a conversation.

If you are using Claude Code or Claude with MCP support, look for a Gmail/Calendar MCP server, connect it through Google's own sign-in flow, and grant only the permissions you are comfortable with (read-only is a safe place to start). Claude then accesses your inbox and calendar through that secure connection, not through stored credentials it can see in plain text.

· · ·

Step 5 — Putting It All Together

Once these four are connected, the pieces work like this: GitHub stores your files, Vercel publishes them as a live website, Notion holds your personal context and notes, and Gmail/Calendar give Claude a window into what is actually happening in your life right now.

I did not connect all four in one weekend. I got GitHub and Vercel working first — just to see a website go live. Notion came a few weeks later. Gmail came after that. Each one only took an afternoon once I stopped trying to do everything at once.

Usage Tips

  • 1Set up one tool at a time. Get GitHub + Vercel working and see a live page before adding Notion or Gmail.
  • 2Start with read-only permissions on Gmail and Calendar connections. Add write access only once you trust the workflow.
  • 3Never commit secrets. API keys and tokens should live in environment variables or a secrets manager — never in a file you push to a public GitHub repo.
  • 4Use a public repo only if you're comfortable with that. Public repos are free on Vercel and GitHub, but keep private information out of them entirely.
  • 5Revoke and rotate immediately if you ever accidentally expose a token or password — every one of these services lets you regenerate keys in seconds.

Q&A

Q: Do I need to pay for any of this?
A: No. GitHub, Vercel, Notion, and Gmail all have free tiers that are more than enough for a personal site and personal AI setup like this one.
Q: Is it safe to make my GitHub repo public?
A: Yes, as long as you never put passwords, API keys, or private personal information directly in the files. Treat anything in a public repo as visible to the whole world, because it is.
Q: What if I get stuck on one of these steps?
A: Take a screenshot of exactly where you are and ask Claude to walk you through just that one step. It's much easier to debug one screen at a time than to try to explain the whole setup at once.
Q: Do I need all four to get started, or can I do just one?
A: Just one is fine. GitHub + Vercel alone gets you a live website. Add Notion when you want Claude to remember more about you. Add Gmail when you want it to act on your real inbox and calendar.
