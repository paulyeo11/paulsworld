TUT04: Build a Claude Investment Agent to Analyze Your Tiger Brokers Portfolio

Step-by-step guide to setting up an AI agent that reads your live portfolio and reports your P&L分步指南：设置AI代理，读取实时投资组合并报告盈亏

Paul Yeo  ·  June 2026  ·  Tutorials Series

My financial dashboard refreshes every morning with live numbers pulled straight from my Tiger Brokers account — positions, profit and loss, and what moved overnight. I built it with Claude, and it did not require me to become a programmer. This guide shows the path I took.

⚠️ Before you start

This guide is about building a tool that reads and summarises your own portfolio data. It is not investment advice, and Claude should never be treated as a licensed financial adviser. You make every buy and sell decision yourself.

· · ·

Step 1 — Get API Access from Tiger Brokers

Tiger Brokers offers an Open API for account holders. Log into your Tiger Trade account, find the Open API / Developer section, and apply for API access. You will receive a Tiger ID, a private key, and account credentials needed to connect programmatically.

🔒 Keep your keys private

Never paste your API key, private key, or account number directly into a chat message or commit it to a public code repository. Store credentials in a local environment file or a secrets manager that only your own machine can read.

· · ·

Step 2 — Choose How Claude Will Connect

There are two practical ways to let Claude read your Tiger data:

Two Ways to Connect

A
A small Python script + Claude Code Use Tiger's official Python SDK to pull positions and P&L into a local file (CSV or JSON). Ask Claude Code to read that file and build you a summary or dashboard. This is the simplest path if you are comfortable running one script.
B
A custom MCP server If you want Claude to query your account live, on demand, a Model Context Protocol (MCP) server wraps the Tiger API as a tool Claude can call directly during a conversation. This takes more setup but means you never have to re-run a script.

I started with option A — a script that pulls my positions every morning into a file — and only added the MCP layer later once I wanted live, on-demand answers.

· · ·

Step 3 — Give Claude the Analyst's Job Description

Once Claude can see your portfolio data, tell it exactly what kind of analysis you want:

You are my portfolio analyst. Each time I share my positions data, summarise: total P&L, biggest winners and losers, percentage in each sector, and anything that looks like a concentration risk. Do not recommend specific trades — flag patterns and ask me clarifying questions instead.

— A starting persona prompt you can copy and edit

This keeps Claude in the role of analyst, not adviser — it surfaces what's happening in your numbers, and you decide what to do about it.

· · ·

Step 4 — Build the Daily Cadence

The real power shows up once this becomes automatic. On my setup, a pipeline runs each morning, refreshes my Tiger positions, and updates my dashboard before I even open my laptop.

Example Daily Routine

🌅
Morning refresh Script pulls latest positions and P&L from the Tiger Open API.
📊
Summary generated Claude reads the refreshed data and writes a short plain-language summary — what moved, what to watch.
📱
You check it Open your dashboard or chat once a day. The numbers are already there, already summarised.

You do not need full automation on day one. Even running the script manually once a day and pasting the output to Claude for a summary gets you most of the benefit.

· · ·

Step 5 — Ask for Insight, Not Just Numbers

Once the basic pipeline works, the real value comes from the questions you ask:

  • "Which 3 positions moved the most this week, and why?"
  • "What percentage of my portfolio is in tech stocks right now?"
  • "Compare this month's P&L to last month."
  • "Am I overly concentrated in any single stock or sector?"

These questions turn a spreadsheet of numbers into something you can actually act on, in plain language.

· · ·

Usage Tips

  • 1Never share raw API secrets in chat. Keep them in a local environment file Claude never sees directly.
  • 2Start manual, automate later. Get the script working reliably before you build a daily schedule around it.
  • 3Ask for summaries, not trade calls. Use Claude to understand your portfolio, not to decide what to buy or sell.
  • 4Double-check the numbers against the official Tiger Trade app periodically, especially after any setup change.
  • 5Keep a simple persona prompt saved so every new chat about your portfolio starts with the same ground rules.
· · ·

Q&A

Q: Is this financial advice?
A: No. This guide is about building a tool that reads and summarises your own data. Any investment decision is yours, and for personalised advice you should speak to a licensed financial adviser.
Q: Can Claude place trades automatically?
A: This guide only covers reading and analysing your portfolio. Giving any AI system the ability to place trades automatically carries real financial risk and is well beyond the scope of a beginner setup — proceed with extreme caution if ever considering it.
Q: Do I need to know how to code?
A: Basic comfort running a script helps for option A, but Claude Code can write and explain most of the script for you. Option B (an MCP server) is more technical and better suited if you already enjoy tinkering.
Q: What if I only have one brokerage account and don't want to write any code?
A: You can still get value by manually exporting a CSV of your positions from the Tiger app and pasting the contents into a Claude chat for a summary — no script required.

Final Thoughts

Building this did not happen in one weekend — I started with a script that just printed my positions, then added a summary, then a dashboard, then automation. Each step was small. Together they replaced a manual process that used to take me twenty minutes every morning with a glance that takes twenty seconds.
