AIT13: Making My IBKR Report Save Itself to Google Drive —
Every Morning at 8am, With No Phone Taps AIT13：让我的IBKR报告自动存进Google Drive——
每天早上8点，手机什么都不用点

Every single day, Interactive Brokers quietly emails me a PortfolioAnalyst report — a PDF with my account's numbers for that day. And every single day, it just sat in my inbox, doing nothing, until I remembered to deal with it. This is the story of how I got Claude to write eleven lines of code that now do that job for me, automatically, at 8am, before I've even picked up my phone. 每一天，Interactive Brokers都会悄悄给我发一封PortfolioAnalyst报告邮件——一份带有当天账户数据的PDF。而每一天，它就那样静静躺在我的收件箱里，什么都没发生，直到我想起要去处理它。这篇文章讲的是，我怎么让Claude写了十几行代码，从此这件事就自动完成了，每天早上8点，在我拿起手机之前就已经做好。

The Problem: A Report I Kept Forgetting问题：一份我总是忘记处理的报告

I already have a rule in my notes: whenever I ask Claude to "check the IBKR folder in Drive," it goes and reads whatever raw data I've saved there and updates my dashboards. That works great — but it depends on me actually putting a file in that folder first. And on most mornings, on my phone, in bed, that's exactly the step I skip.我的笔记里其实已经有一条规矩：每当我让Claude「查一下Drive里的IBKR文件夹」，它就会去读取我存在那里的原始数据，并更新我的仪表板。这个办法很好用——但前提是我得先把文件放进那个文件夹。而大多数早晨，我人在床上，用手机操作，恰恰就是这一步，我经常忘了做。

Meanwhile, IBKR was already sending me exactly the data I needed, every single day, straight to my Gmail — a "PortfolioAnalyst Report" PDF, sent by donotreply@interactivebrokers.com, usually sometime mid-morning Singapore time. It was right there. I just never moved it anywhere useful.与此同时，IBKR其实早就每天把我需要的数据直接发到我的Gmail里了——一份「PortfolioAnalyst Report」PDF，由donotreply@interactivebrokers.com发送，通常是新加坡时间的上午中段。数据其实一直都在那里，只是我从没把它挪到有用的地方去。

The Fix: Let Gmail Talk to Drive Directly解决办法：让Gmail直接和Drive对话

The answer wasn't a new app or a paid subscription. Google already lets Gmail and Google Drive run a small script for you, for free, on a timer — it's called Google Apps Script. I asked Claude to write one that does exactly one job: look for today's IBKR report email, grab the PDF attached to it, and drop it straight into my existing "IBKR" Drive folder — the same one my dashboards already read from.答案不是什么新App，也不用花钱订阅。Google本身就能让Gmail和Google Drive免费按时间表运行一小段脚本——这就是Google Apps Script。我让Claude写了一个，它只做一件事：找到当天的IBKR报告邮件，取出里面的PDF附件，直接存进我原本就有的那个「IBKR」Drive文件夹——也就是我的仪表板一直在读取数据的那个文件夹。

function copyIBKRReportToDrive() { var folder = DriveApp.getFolderById(IBKR_DRIVE_FOLDER_ID); var threads = GmailApp.search( 'from:donotreply@interactivebrokers.com ' + 'subject:"PortfolioAnalyst Report" has:attachment newer_than:2d' ); threads.forEach(function (thread) { thread.getMessages().forEach(function (message) { message.getAttachments().forEach(function (attachment) { var already = folder.getFilesByName(attachment.getName()); if (!already.hasNext()) folder.createFile(attachment); }); }); }); }

That's the whole thing. No servers, no monthly fee, nothing to install on my phone — it lives entirely inside my Google account.就这么多，没有服务器，不用每月付费，手机上也不需要安装任何东西——它完全运行在我的Google账户内部。

What the script actually checks这段脚本实际做的事
  • Searches Gmail for mail from IBKR's report sender, with the exact subject line, from the last 2 days — so it never misses a day even if it runs a little late.在Gmail里搜索来自IBKR报告发件人、带有确切主题的邮件，范围是最近2天——这样即使运行得稍晚一点，也不会漏掉哪一天。
  • Checks the Drive folder first, by filename, before saving — so re-running it never creates duplicate copies of the same day's report.保存之前会先按文件名检查Drive文件夹里是否已存在——这样即使重复运行，也不会把同一天的报告存成好几份。
  • Saves straight into the same "IBKR" folder my existing dashboard workflow already reads from — nothing else in that setup had to change.直接存进我原本仪表板流程一直在读取的那个「IBKR」文件夹——这套流程里其他任何东西都不需要改动。

Setting the Clock: 8am, Singapore Time设定时间：新加坡时间早上8点

Writing the script is only half the job — it also needs a clock. In Apps Script, that's a "trigger": a rule that says "run this function automatically, on this schedule." I set mine to a daily Day Timer between 8am and 9am, with the project's time zone set to Asia/Singapore.写脚本只完成了一半——它还需要一个时钟。在Apps Script里，这叫做「触发器」：一条规则，告诉系统「按这个时间表自动运行这个函数」。我把它设成了每天的Day Timer，时间段是早上8点到9点，项目的时区设为Asia/Singapore（新加坡）。

⚠️

Honest limitation, not a bug: Google's day-timer triggers fire once somewhere inside the hour window you pick, not at the exact minute. "8am" here means "sometime between 8:00 and 9:00am" — close enough for a daily report, but worth knowing if you ever expect it on the dot.老实说的一个限制，不是漏洞：Google的Day Timer触发器，只会在你设定的那一小时窗口内的某个时间点运行一次，而不是精确到那一分钟。这里的「早上8点」其实是「早上8点到9点之间的某个时刻」——对于每日报告来说已经够用了，但如果你期待它分秒不差，就得留意这一点。

Why It Had to Be Drive, Not Just the Inbox为什么一定要存到Drive，而不是留在收件箱

I could have just left the email alone and told Claude to search my inbox directly each time. But my standing workflow — the one my dashboards and Claude's routine already trust — is built around one specific Drive folder, checked by phrase: "check the IBKR folder in Drive." Feeding the report into that same folder means nothing downstream has to change. It's not a parallel system; it's the same door, just one that now opens itself.其实我也可以什么都不做，直接让Claude每次都去我的收件箱里搜索。但我原本的固定工作流——我的仪表板和Claude的日常操作已经在依赖的那一套——是围绕着一个特定的Drive文件夹搭建的，触发方式是那句话：「查一下Drive里的IBKR文件夹」。把报告存进同一个文件夹，意味着后面的一切都不需要改动。这不是另外搭了一套系统，而是同一扇门——只是现在它会自己打开。

The best automation isn't the one that adds a new habit to remember. It's the one that quietly does the habit I already had — the one I kept forgetting on my own.最好的自动化，不是让人多记一件新事。而是安安静静地替我完成那件我本来就该做、却总是自己忘记的事。

How I Set It Up我是怎么设置的

Open script.google.com and start a new project.打开script.google.com，新建一个项目。

This runs inside my own Google account — no new sign-up, no third-party app given access to my email.这就在我自己的Google账户里运行——不需要注册新账号，也不需要把邮箱权限交给任何第三方App。

Paste in the script and set the time zone.粘贴脚本，并设置时区。

Project Settings → time zone → Asia/Singapore, so "day timer" actually means my morning, not California's.项目设置→时区→Asia/Singapore（新加坡），这样「Day Timer」指的才是我这边的早上，而不是加州的早上。

Run it once by hand, and approve the permissions.手动运行一次，并批准权限。

Google asks to confirm the script may read Gmail and write to Drive — a one-time approval, only for this script.Google会要求确认这个脚本可以读取Gmail并写入Drive——只需批准一次，而且只针对这个脚本。

Add the daily trigger and walk away.添加每日触发器，然后就不用管了。

Triggers → Add Trigger → Time-driven → Day timer → 8am–9am. From that morning on, the PDF is already waiting in Drive before I've unlocked my phone.触发器→添加触发器→按时间驱动→Day Timer→8点至9点。从那天早上起，我还没解锁手机，PDF就已经在Drive里等着我了。

✅

The part I like best: it fits into the exact routine I already had, instead of asking me to learn a new one. "Check the IBKR folder in Drive" still means exactly what it always meant — the folder just fills itself now.我最喜欢的一点：它完全嵌入了我原本就有的那套习惯，而不是要我再学一套新的。「查一下Drive里的IBKR文件夹」这句话的意思一点都没变——只是现在，那个文件夹会自己把东西准备好。
