AI3: One Week That Changed Everything

What I learned building my own website with AI — as a 70-year-old retiree with no coding background.

✍️ Paul Yeo 📅 May 2026 ⏱ 5 min read

"I spent one week building a website with AI. I expected to learn a little technology. What I discovered was a whole new way of thinking."

1

I Now Know How to Use AI Better

The secret: give AI more context, get far better results.

Before this week, I used AI like a search engine — type a question, read an answer, move on. Now I understand that AI is a collaborator. The more context and detail you give it, the better the result.

I learned to give AI the raw link to my code. Once I did that, it could read the actual file, understand exactly what was wrong, and fix it. That was a turning point.

Key insight: AI is not a vending machine. It is a thinking partner. The quality of your output depends on the quality of your input.
2

AI Can Troubleshoot Like a Real Engineer

AI reads your actual code — it is not guessing.

My view counter stopped working. I had no idea why. I tried refreshing, guessing, worrying. Then I gave Claude the raw link to my page and asked it to look.

Within seconds it identified the exact line causing the problem, explained why it failed, and gave me the corrected code. What might have taken me days to figure out was solved in minutes.

The lesson: Once you provide the raw link, AI can SEE the code. It is not guessing — it is reading. This changes everything about how you ask for help.
3

GitHub & Vercel — Simpler Than I Thought

GitHub is just a filing cabinet in the cloud. Vercel turns it into a live website.

I expected these tools to be for programmers only. I was wrong. GitHub is simply a place to store your files online. Vercel is simply a way to publish those files as a live website. That is the whole thing.

  • 📁GitHub stores my HTML files — it is my filing cabinet in the cloud.
  • 🚀Vercel publishes the files — every time I update GitHub, the live site updates automatically.
  • 🔗The raw link from GitHub is what I give to AI — so it can read my actual code.
4

My iPhone Is All I Need for Maintenance

Small updates, anywhere, anytime — just a phone is enough.

This surprised me the most. I do not need to sit at a desktop computer to manage my website. With my iPhone, I can open GitHub, edit a file, save it — and the website updates live.

For small updates — fixing a typo, adding a new article link, changing a date — my phone is completely sufficient. Technology has become truly mobile.

Practical tip: Use the GitHub mobile app. It lets you edit files directly on your phone. Commit the change and Vercel redeploys within seconds.
5

AI Is Just Like a Human — Give It Context

Show AI the actual file — like showing a knowledgeable friend.

When I told a human friend "my counter is broken," they could not help because they could not see the code. AI works the same way. If I say "something is wrong," AI can only guess.

But once I provide the raw link — the actual file — AI can see everything. It is exactly like showing the file to a knowledgeable friend and saying "look at this."

  • ❌Vague: "My page doesn't work, can you fix it?"
  • ✅Specific: "Here is the raw link: [link]. The view counter shows 0 instead of the real count."
6

Write Your Preferences — Save Hours Every Time

Write it once in your preferences — AI remembers it forever.

Every time I started a new conversation with AI, I had to re-explain everything: my website URL, my rules, my preferences. It was exhausting and repetitive.

Then I discovered the User Preferences setting. Now everything I want AI to always remember — my live site URL, my rules about view counters, image handling — is written there once and applies automatically to every conversation.

The result: Every new article I ask Claude to build already includes the Home button, view counter, and correct image paths — automatically, without me repeating myself.

One Week. Six Lessons. Zero Regrets.

At 70, I built a live website with multiple articles, a view counter, and a popular stories ranking — all from my phone and laptop, with no prior coding knowledge. If I can do it, you can too.
